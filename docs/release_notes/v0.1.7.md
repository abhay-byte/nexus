## Changelog

### Major Features

- **feat: Overhauled Add Project dialog**
  - Removed Default Agents selection (now configured per-project in Settings)
  - Added **Project Category** selector (Web, App, Game, API / Backend, ML / AI, CLI Tool, Other)
  - Added **AI Agency** toggle + personality selection with upstream catalog fetch
  - Added **MCP Presets** selector (context-mode, context7, playwright, github, filesystem, sequential-thinking, android-mcp)
  - Added **Spec Kit** bootstrap toggle + agent selector (Codex, Claude Code, Gemini CLI)
  - Added **Caveman** one-click install for supported agents (Claude Code, Gemini CLI, Cline, Kiro)
  - Post-creation automation: agency sync, spec-kit bootstrap, and caveman install run automatically after project creation

- **feat: Project category context in AGENCY.md**
  - When syncing an agency agent, if a project category is set, the generated `AGENCY.md` now prepends a **Project Context** block with tailored instructions for the domain (web, app, game, api, ml, tool)

- **feat: Right-click file context menu in Project Directory**
  - Right-click any file in the project tree to:
    - **Open in Editor** — opens the file with the system's default application
    - **Preview** — opens a modal with syntax-highlighted file contents for text files

- **feat: Auto-bump running projects to top**
  - Projects with running agents automatically move to the top of the sidebar list

- **feat: Drag-and-drop project reordering**
  - Drag projects in the sidebar to reorder them persistently
  - Works in both expanded and collapsed sidebar modes

### Windows Fixes

- **fix: Console window no longer appears on Windows**
  - Changed `windows_subsystem` attribute to unconditional on Windows (previously only hid console in release builds)
  - Double-clicking the `.exe` now opens only the GUI window

- **fix: WebView2 auto-installation**
  - Added `embedBootstrapper` to the Windows bundle config
  - The installer now silently downloads and installs WebView2 Runtime if missing on the target machine
  - Fixes "WebKitWebProcess2 not found"-type startup crashes caused by missing WebView2

- **fix: NSIS installer supports per-user and system-wide installs**
  - `installMode: "both"` allows installation without admin privileges by default

### UI Fixes

- **fix: Agency dropdown in Add Project dialog now matches design system**
  - Replaced native `<select>` with the same brutalist custom dropdown used in Settings
  - Features: border-4 trigger, yellow category badge pill, rotating expand_more icon, offset shadow panel, selected highlight in #ffcc00
